Comments & Technical Details

Overall Structure

The site contains four pages: index(home), past, future, and comments. Navigation between them is handled via HTML anchor tags inside a navigation bar that appears on the top for all the pages for consistency and efficient user interface.

Internal Page Structure

Each page uses semantic HTML5 tags including <nav>, <main>, <div>, <footer>. The layout is kept simple using Flexbox for responsive alignment, and content is seperated using divs and semantic groupings.

CSS Styling

All styles are defined in websystems.css. I used a pastel gradient background, a custom Google font ("Press Start 2P"), and a soft animated loading screen. I've also included glowing animations, hover effects and rounded corners for an aesthetic retro feel.

Graphic Design

I chose a cute, softcore, gamer aesthetic with pastel colours and a retro 2000s video game theme as my sister suggested it. Pixel fonts and images help create a cohesive vibe. Legibility and layout were important, so I used spacing, simple navigation, and clean structure.

Accessibility Features

Keyboard accessibility is an important part of the user experience. Affecting the experience of many users, mainly people with motor disabilities — any condition that limits movement or coordination.

All images include alt text. I ensured the layout is readable without CSS by setting width and height in HTML and avoided JavaScript altogether as per assignment requirements.

I also utilised a:focus, button:focus in a bold contrasting colour #40e0d0 to make user selection legible (particularly Tab& Tab + Shift menu slection).

Disabling CSS

If CSS is disabled, the site remains functional and readable. I added sizing directly into HTML image tags so they don’t overflow. The structure remains in place with meaningful text and alt descriptions.
